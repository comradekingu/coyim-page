Otrv3 Authenticated Key Exchange (AKE)


What is an Authenticated Key Exchange?

Key establishment is a process by which two (or more) parties establish a shared secret. This key is later used to achieve some goal such as confidentiality or data integrity.

There are two kind of key establishment protocols:

  • Key Transport: the key is created by one entity and securely transmitted to a second entity.
  • Key Agreement: a shared key is created by information given by both parties who jointly created it.

A Key Agreement is usually done by two or more honest parties: parties that can execute the steps of a given protocol correctly. This agreement is usually referred to be "authenticated" if both of those entities are identified and either of them know that no other entity can learn the value of the shared secret key. The honest parties, therefore, have confidence of the identity of the party they are communicating with. This is usually referred as "Authenticated Key Agreement" (AK) protocol


How is AKE used in OTRv3

OTRv3 achieves AKE by doing a Diffie Hellman key exchange with anonymous authetication. It is based on Direct Anonymous Attestation (DAA) and the SIGMA family of key exchange protocols (as used in the Internet Key Exchange [IKE] standard).

For the Diffie-Hellman key exchange, OTRv3 uses a particular group (1536-bit MODP group) of id 5 as defined here .

The general idea is that Alice and Bob do an unauthenticated Diffie-Hellman (D-H) key exchange to set up an encrypted channel, and then do mutual authentication inside that channel.
